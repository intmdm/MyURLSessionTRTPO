# 1 Введение

<a name="appointment"/>

## 1.1 Назначение
В этом документе описаны функциональные и нефункциональные требования к приложению «Currency» для ОС IOS. Этот документ предназначен для команды, которая будет реализовывать и проверять корректность работы приложения. 

<a name="business_requirements"/>

## 1.2 Бизнес-требования

<a name="initial_data"/>

### 1.2.1 Исходные данные
Нужно разработать приложение позволяющее оперативно и быстро оценить курс валют с наглядным графиком изменения стоимости и кросс-курсом для доллара.
<a name="business_opportunities"/>

### 1.2.2 Возможности бизнеса
Крайне простое, удобное и приятное в использовании приложения для мониторинга курса валют. Умеет строить удобную в работе диаграмму.
<a name="project_boundary"/>

### 1.2.3 Границы проекта
Приложение имеет один большой график по дням, на котором представлен курс валюты.

## 1.3 Аналоги
Обзор аналогов представлен в документе [Overview of analogues](../Requirements/Overview%20of%20analogues.md).

## 3.1 Функциональные требования

<a name="main_functions"/>

### 3.1.1 Основные функции

<a name="enter_to_app"/>

#### 3.1.1.1 Вход пользователя в приложение
**Описание.** Пользователь имеет возможность использовать приложение без создания собственного профиля.

| Функция | Требования | 
|:---|:---|
| Вход в приложение без создания собственного профиля | Приложение должно предоставить пользователю возможность войти в приложение анонимно |

<a name="displays_all_tasks_already_added"/>

#### 3.1.1.2 Отображение всех уже добавленных задач
**Описание.** После входа пользователя в приложение отображается обновленный график валют.

| Функция | Требования | 
|:---|:---|
| Загрузка данных | После входа пользователя в приложение отображается график |

<a name="restrictions_and_exclusions"/>

### 3.1.2 Ограничения и исключения
1. Нет возможности изменить валюту.
2. Нет возможности изменить банк.
